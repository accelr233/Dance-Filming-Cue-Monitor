# Dance Filming Cue Monitor

手机端舞蹈拍摄提示屏。它支持两种同步方式：

- 手动同步：现场音乐开始或第一拍落下时点“同步开始”。
- 自动听歌同步：先上传原曲生成指纹，再用手机麦克风监听现场音响，识别当前音乐位置并自动校准 cue 时间轴。

## 本地启动

在 PowerShell 里运行：

```powershell
.\start-monitor.ps1
```

脚本会启动一个本地网页服务器，并显示电脑和手机可访问的地址。手机和电脑需要连在同一个 Wi-Fi。

启动后保持 PowerShell 窗口打开；拍完后按 `Ctrl+C` 关闭服务。

现在启动脚本默认使用 HTTPS，例如：

```text
https://192.168.0.184:5173/
```

手机端要打开麦克风，需要把本地 CA 证书安装并设为信任：

```text
.cert/local-ca.crt
```

第一次运行启动脚本时会自动生成这个证书文件。把它传到手机上安装并信任后，再用脚本显示的 HTTPS 手机地址打开页面。

## 使用自动同步

1. 打开页面。
2. 在“自动听歌同步”里上传要播放的原曲音频。
3. 等页面显示“已生成指纹”。
4. 点“开始监听现场”，允许浏览器使用麦克风。
5. 现场音响从任意位置播放时，页面会尝试识别当前时间，例如 `00:37.0`，并把 cue 时间轴跳到那里。
6. 再点“停止监听”可以关闭麦克风监听。

## 重要限制

手机浏览器打开麦克风要求安全上下文，也就是 HTTPS 或 localhost。本项目已经提供本地 HTTPS 服务，但手机必须信任 `.cert/local-ca.crt`，否则浏览器仍可能拒绝麦克风权限。

这个版本是本机浏览器端的轻量音频指纹原型，适合舞蹈音乐、现场音响比较清晰的场景。环境噪声很大、手机离音响太远、音乐被严重变速或升降调时，匹配会变慢或不准。

## 批量格式

```text
00:04, 开场全景, 舞台全景
00:12, A组, 左侧中景
00:24, 小林, 特写
```

提示会保存在当前浏览器的本地存储里。
